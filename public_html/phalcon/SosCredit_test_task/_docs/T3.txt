Тестовое задание:
• Написать простой CRUD интерфейс, который включает грид и форму
• Используя php framework phalcon 2 (https://phalconphp.com/ru/)
• Используя php 5.6, nginx, php-fpm
• Используя в качестве БД - Mysql желательно PostgreSql;
• Интерфейс на Twitter Bootstrap 3;
• Используя javascript framework jquery
• Добавление через ajax;
• Авторизации и прочего НЕ нужно, только write\read\remove
 
Детали:
Имеется список ранее добавленных сообщений с постраничной разбивкой (10 на стр) и датой добавления.
Форма состоит из полей Имя, Телефон, Почта, Сообщение.
Каждое поле должно быть провалидированно. Имя не меньше 3 символов, Телефон - стандартный формат телефонов, Почта - валидация email, Сообщение не пустое и не больше 200 символов.
После ввода сообщения и нажатия кнопки "добавить" происходит сохранение сообщения в БД, а в интерфейсе происходит добавление этого сообщения над ранее добавленными.
В гриде у каждой сообщения есть ссылки , редактировать и удалить.
У грида есть фильтры по имени, телефону и почте.
Логика не должна быть в контроллере, только получение параметров с request, вызов модели получение данных из нее, передача данных во view.
 
Форма это отдельный класс,
логика работы грида это отдельный класс, где обрабатываются параметры по сортировки и фильтрам отдаются в модель работы с базой,
должно получиться, как минимум три класса, класс модели по работе с таблице в базе данных, класс формы сохранения, редактирования,
и класс грида, который знает как правильно инициализировать модель таблицы, как построить запрос в метод find, получает данные преобразовывает их в обычный массив и возвращает в нужном и простом виде для вывода их во view.
